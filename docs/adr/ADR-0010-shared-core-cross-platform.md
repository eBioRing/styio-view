# ADR-0010: Desktop And Mobile Share A Core Framework

**Purpose:** 记录 `styio-view` 在桌面端与移动端之间共享核心框架、数据模型和服务层，而只在展示与交互层分化的决定。

**Last updated:** 2026-04-12

**Status:** Accepted

**Date:** 2026-04-12

## Context

产品希望：

1. 电脑端与手机端尽量共享工程基础
2. 后续只针对手机端展示界面和渲染逻辑做优化
3. 不把移动端做成完全独立产品

## Decision

共享以下核心层：

1. 文档模型
2. 语言服务接口
3. 模块宿主与 capability matrix
4. 运行事件协议
5. 主题与 profile 数据模型

分端实现以下层：

1. 页面布局
2. 手势与输入模型
3. 局部渲染调优策略
4. 平台执行入口

## Alternatives

1. 为移动端重做一套完全独立框架：共享率过低，维护成本高。
2. 强行把桌面交互和布局完整复用到手机端：可用性差。

## Consequences

1. 共享核心可减少语言、运行时和模块系统的重复实现。
2. 移动端仍保留充分的展示与交互优化空间。
3. 任何移动端优化不得破坏共享核心数据模型。
