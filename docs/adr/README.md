# ADR Docs

**Purpose:** 定义 `docs/adr/` 的范围和命名规则；具体 ADR 清单见 [INDEX.md](./INDEX.md)。

**Last updated:** 2026-04-12

## Scope

1. 只记录会影响长期实现路径的架构决策。
2. 每个 ADR 只回答一个主问题，并记录背景、决策、替代方案和后果。
3. 里程碑级临时任务拆解不写在 ADR 内，留在 `docs/milestones/`。

## Naming Rules

1. 文件名使用 `ADR-XXXX-<slug>.md`。
2. 标题内需写明 ADR 编号与短标题。
3. 文首必须包含 `Purpose`、`Last updated`、`Status`、`Date`。
